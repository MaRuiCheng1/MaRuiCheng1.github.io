1.序言

1.1 blog创建契机

关于这个blog的创建契机，是在毕业设计期间，我有幸观看到了许多学长学姐的个人blog。这让我十分惊讶与羡慕，因为对于我而言，能拥有一个自己的发癫网站实在是太帅了，鉴于此，我也决定制作一个属于自己的blog用于发表和存储一些自己撰写的论文、日常生活中的吐槽信息、打游戏的战绩图等等（哈哈哈哈哈），当然我也会用这个平台，记录与朋友间的快乐生活时光。

1.2 blog编写相关

blog编写采用markdown这一类轻量级语言，初次接触我便能感受到其在创作过程中的便携性：它可以容易的导成html、word、图像、PDF等文件文档，在编写中也能通过其语言格式完成一些超链接、表格、数学公式和图片内容插入等（目前我还未通过图床完成图片转网址，因此未在文内插入相关图片内容）。太多编写案例我先存在这，以后哪天我写着写着忘了再来这里找找看怎么写（www）。

2.关于

2.1 现状

目前（截止2023年11月28日），网页搭建还不是很完善：评论区、图片库、我心目中的二次元主题和一个可以实时交互的虚拟形象均为完成部署，我也会在接下来的时间里尽快完成搭建，让这个网站看起来更加美丽生动。之后每天我都会上传当天我的搭建进度到这篇文章里，我也会把搭建过程中遇到的问题都写下来，方便有兴趣也来搭建blog的大家查看和避雷。

2.2 每日记录

2023年11月

28日

昨晚在完成了网页搭建以后，由于没有使用图床功能，我选择通过git上传本地图片到github库。由于对github知之甚少，不知道其对所接受文件有要求（单个文件不可大于100mb），接收到了以下报错信息：
remote: error: GH001: Large files detected. You may want to try Git Large File Storage
处理方法：删除根目录下.deploy_git文件夹与public文件夹，在终端使用hexo cl;hexo g;hexo d重新配置网站即可。

由于使用git上传时会出现被墙的情况，因此大家在上传之前记得在git中输入ssh -T git@github.com查看git与github的通信连接情况，如果连接不成功就使用魔法（懂得都懂）。
